# 🚀 Быстрый старт: Этапы 1+2

---

## 📋 Шаг 1: Настройка Backend

### 1.1 Создайте файл `.env`

```bash
cd backend
cp .env.example .env
```

### 1.2 Откройте `.env` и заполните данные InfluxDB

```env
# InfluxDB (ОБЯЗАТЕЛЬНО!)
INFLUX_URL=http://localhost:8086
INFLUX_TOKEN=ваш_токен_influxdb
INFLUX_ORG=ваша_организация

# JWT секрет (можно оставить как есть для теста)
JWT_SECRET=your-secret-key-min-32-chars-long-please-change-this
```

> **Где взять INFLUX_TOKEN?**
> 1. Откройте InfluxDB UI: http://localhost:8086
> 2. Settings → Tokens → Generate Token
> 3. Скопируйте токен в `.env`

### 1.3 Установите зависимости

```bash
npm install
```

### 1.4 Запустите backend

```bash
npm run dev
```

✅ Backend запущен на `http://localhost:3001`

## 🎯 Шаг 3: Тестирование

### 3.1 Авторизация

1. Откройте http://localhost:5173/login
2. Войдите как **admin / admin123**

### 3.2 Страница "Станции"

1. Нажмите вкладку **"Станции"** (доступно только админам)
2. Увидите 2 предустановленные станции:
   - **LOGO Лаборатория**
   - **LOGO Цех 1**

### 3.3 Создайте тестовую станцию

1. Нажмите **"Добавить"**
2. Заполните форму:
   - Название: **Моя станция**
   - Bucket: **telemetry**
   - Measurement: **mylab**
3. Нажмите **"Сохранить"**

### 3.4 Добавьте метрику

1. Выберите созданную станцию
2. В правой колонке нажмите **"Добавить"**
3. Заполните форму:
   - Название: **Температура**
   - Field: **температура**
   - Channel: **AI1**
   - Unit: **°C**
4. Нажмите **"Сохранить"**

### 3.5 Главная страница

1. Перейдите на вкладку **"Главная"**
2. Проверьте:
   - ✅ Загружаются станции из БД
   - ✅ Отображаются метрики
   - ✅ Работает фильтрация
   - ✅ Данные обновляются автоматически

## 🐛 Возможные проблемы

### Backend не запускается

**Проблема:** `Error: Переменная окружения INFLUX_TOKEN не установлена`

**Решение:** Проверьте файл `.env` - все переменные заполнены?

---

### InfluxDB недоступен

**Проблема:** `InfluxDB недоступен`

**Решение:**
1. Проверьте что InfluxDB запущен: http://localhost:8086
2. Проверьте `INFLUX_URL` в `.env`
3. Проверьте токен доступа

---

### Нет данных на графиках

**Проблема:** Графики пустые

**Решение:**
1. Убедитесь что в InfluxDB есть данные
2. Проверьте что bucket/measurement совпадают
3. Попробуйте выбрать больший временной интервал
