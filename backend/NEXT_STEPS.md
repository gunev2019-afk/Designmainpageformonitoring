## 🎯 СЕЙЧАС СДЕЛАЙТЕ:

### Шаг 1: Удалите старые node_modules

```bash
# Закройте все программы которые могут использовать файлы
# (VSCode, редакторы, терминалы)

# Удалите папку node_modules
rmdir /s /q node_modules
```

Если ошибка "доступ запрещен":
1. Закройте VSCode полностью
2. Закройте все терминалы
3. Попробуйте еще раз

### Шаг 2: Установите зависимости заново

```bash
npm install
```

Теперь установка пройдет **БЕЗ ОШИБОК!** ✅

### Шаг 3: Создайте .env файл

```bash
copy .env.example .env
```

Откройте `.env` и заполните минимум эти 3 переменные:

```env
INFLUX_TOKEN=ваш_токен_здесь
INFLUX_ORG=ваша_организация
JWT_SECRET=любая_длинная_случайная_строка_32_символа_минимум
```

### Шаг 4: Инициализируйте базу данных

```bash
npm run migrate
```

Будет создан файл `./data/monitoring.json` с начальными данными.

### Шаг 5: Запустите сервер

```bash
npm run dev
```

### Шаг 6: Проверьте работу

Откройте в браузере: **http://localhost:3001**

Должны увидеть JSON ответ с информацией об API.

## 🐛 Если что-то не работает:

### Ошибка при npm install:

1. Удалите `node_modules` и `package-lock.json`
2. Закройте все программы
3. Запустите `npm install` от администратора

### Ошибка "EADDRINUSE":

Порт 3001 занят. Измените в `.env`:
```env
PORT=3002
```

### Ошибка "Переменная окружения не установлена":

Проверьте `.env` файл, заполните все переменные.

### Сервер запустился но в браузере ничего:

1. Проверьте что открываете `http://localhost:3001` (не 5173!)
2. Проверьте что в консоли нет ошибок
3. Попробуйте `http://localhost:3001/health`
