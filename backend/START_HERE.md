## ⚡ БЫСТРЫЙ СТАРТ (3 минуты)

### 1. Установка зависимостей:
```bash
npm install
```

### 2. Настройка окружения:
```bash
cp .env.example .env
```

Откройте `.env` и заполните **ЭТИ 3 ПЕРЕМЕННЫЕ**:

```env
INFLUX_TOKEN=ваш_токен_из_influxdb    # ОБЯЗАТЕЛЬНО!
INFLUX_ORG=ваша_организация           # ОБЯЗАТЕЛЬНО!
JWT_SECRET=случайная_строка_32_символа # ОБЯЗАТЕЛЬНО!
```

### 3. Проверка настройки:
```bash
npm run check
```

Если все ✅ зеленое - идем дальше!

### 4. Инициализация БД:
```bash
npm run migrate
```

### 5. Запуск:
```bash
npm run dev
```

### 6. Проверка:
Откройте http://localhost:3001

Должны увидеть JSON с информацией об API.

✅ **ГОТОВО!** Backend работает!

## 🐛 ЧАСТЫЕ ПРОБЛЕМЫ

### "Переменная окружения не установлена"
➡️ Заполните `.env` файл (особенно INFLUX_TOKEN, INFLUX_ORG, JWT_SECRET)

### "EADDRINUSE: порт занят"
➡️ Измените `PORT=3002` в `.env`

### "Cannot find module"
➡️ Запустите `npm install`

### Ошибка компиляции TypeScript
➡️ Проверьте что установлен TypeScript: `npm install -g typescript`

## 🎓 ОБУЧЕНИЕ

Если вы новичок в backend разработке:

1. **Express.js** - веб-фреймворк
   - Роуты: определяют endpoints API
   - Middleware: обрабатывают запросы
   - Controllers: содержат логику

2. **TypeScript** - JavaScript с типами
   - Интерфейсы: описывают структуру данных
   - Типы: предотвращают ошибки

3. **SQLite** - легкая база данных
   - Таблицы: хранят данные
   - Запросы: получают/изменяют данные

4. **InfluxDB** - база данных для временных рядов
   - Bucket: контейнер данных
   - Measurement: тип измерения
   - Field: конкретное значение
   - Tag: метка для фильтрации
