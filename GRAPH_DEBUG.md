# 🔍 ДИАГНОСТИКА ПУСТЫХ ГРАФИКОВ

## 📋 ЧТО ДЕЛАТЬ:

### 1. Обновите страницу (Ctrl+Shift+R)

### 2. Откройте консоль (F12)

### 3. Найдите блок с логами графика:

```
📊 ========== MonitoringChart START ==========
📊 Входные параметры: {...}
...
📊 ========== MonitoringChart END ==========
```

---

## 🎯 ПРОВЕРЬТЕ ПО ПОРЯДКУ:

### ✅ Шаг 1: Есть ли данные?

Ищите строку:
```
📊 Входные параметры: {dataLength: XX, ...}
```

**❌ ПРОБЛЕМА А: dataLength: 0**
```
📊 Входные параметры: {dataLength: 0, ...}
```

**Причина:** Данные не загружены из InfluxDB

**Решение:** Смотрите выше в логах:
```
✅ Данных получено: 0 записей
```

**→ В InfluxDB нет данных!** 

**Как проверить:**
1. Откройте http://localhost:8086
2. Data Explorer
3. Выполните запрос:
```flux
from(bucket: "telemetry")
  |> range(start: -1h)
  |> filter(fn: (r) => r._measurement == "lab")
  |> limit(n: 10)
```

**Если пусто** → Нужно добавить данные в InfluxDB
**Если есть данные** → Проверьте настройки станции (bucket, measurement)

---

### ✅ Шаг 2: Есть ли метрики?

Ищите строку:
```
📊 Фильтрация метрик:
  - Всего метрик в базе: XX
  - Видимые имена метрик: [...]
  - Отфильтровано объектов метрик: XX
```

**❌ ПРОБЛЕМА Б: Отфильтровано объектов метрик: 0**

```
📊 Фильтрация метрик:
  - Всего метрик в базе: 3
  - Видимые имена метрик: ["Температура канал 1", ...]
  - Отфильтровано объектов метрик: 0
⚠️ НЕТ ВИДИМЫХ МЕТРИК! Проверьте:
  - metrics: ["Другое название 1", "Другое название 2"]
  - visibleMetrics: ["Температура канал 1", ...]
```

**Причина:** Имена метрик НЕ СОВПАДАЮТ!

**В базе:** `"Другое название 1"`  
**Ищем:** `"Температура канал 1"`

**Решение:**
1. Перейдите на страницу **Станции**
2. Проверьте **display_name** у метрик
3. Убедитесь что галочка **"Показывать на графике"** (show_in_chart) стоит

---

### ✅ Шаг 3: Правильно ли разделены метрики?

Ищите строки:
```
📊 Разделение по типам:
  🌡️ Температурных метрик: X [...]
  💧 Влажностных метрик: Y [...]
```

**❌ ПРОБЛЕМА В: Температурных метрик: 0, Влажностных метрик: 0**

```
📊 Разделение по типам:
  🌡️ Температурных метрик: 0 []
  💧 Влажностных метрик: 0 []
```

**Причина:** Метрики есть, но у них неправильный `unit` или `field`

**Проверьте детали:**
```
  - Детали видимых метрик: [
    {id: 1, display_name: "...", field: "???", unit: "???", ...}
  ]
```

**Температура должна иметь:**
- `unit: "°C"` **ИЛИ**
- `field` содержит "температ" или "temp"

**Влажность должна иметь:**
- `unit: "%"` **ИЛИ**
- `field` содержит "влажн" или "humid"

**Решение:**
1. Перейдите **Станции** → выберите станцию
2. Проверьте метрики:
   - **Единица измерения** (unit): `°C` или `%`
   - **Поле в InfluxDB** (field): должно содержать правильное слово

---

### ✅ Шаг 4: Есть ли значения в данных?

Ищите строку:
```
📊 Сырые данные (первые 3 записи):
  [0]: {timestamp: "...", metric_1: 22.5, metric_2: 23.1, ...}
  [1]: {timestamp: "...", metric_1: 22.3, metric_2: 23.0, ...}
```

**❌ ПРОБЛЕМА Г: В данных НЕТ metric_X**

```
📊 Сырые данные (первые 3 записи):
  [0]: {timestamp: "2026-03-17T12:30:00Z"}
  [1]: {timestamp: "2026-03-17T12:31:00Z"}
```

**Причина:** Backend не добавил значения метрик в данные

**Проверьте:**
1. F12 → Network → Найдите запрос к `/api/history`
2. Посмотрите Response:
```json
{
  "success": true,
  "data": [
    {"timestamp": "...", "metric_1": 22.5}
  ]
}
```

**Если в Response нет `metric_X`:**
→ Проблема в backend! Проверьте backend/src/routes/data.ts

---

### ✅ Шаг 5: Правильно ли форматированы данные?

Ищите строку:
```
  📊 Форматированная запись [0]: {time: "...", rawTime: "...", metric_1: 22.5, ...}
  📊 Форматированная запись [1]: {time: "...", rawTime: "...", metric_1: 22.3, ...}
```

**✅ Если видите metric_X с числами** → Данные готовы для графика!

**❌ Если metric_X: null** → Проблема в сырых данных (см. Шаг 4)

## 🔍 ТИПИЧНЫЕ ПРОБЛЕМЫ:

### 🔴 Проблема 1: Нет данных в InfluxDB

**Логи:**
```
✅ Данных получено: 0 записей
📊 Входные параметры: {dataLength: 0, ...}
```

**Решение:**
1. Откройте InfluxDB → http://localhost:8086
2. Добавьте тестовые данные через Data Explorer

---

### 🔴 Проблема 2: Имена метрик не совпадают

**Логи:**
```
⚠️ НЕТ ВИДИМЫХ МЕТРИК!
  - metrics: ["Temp AI1", "Temp AI2"]
  - visibleMetrics: ["Температура канал 1", "Температура канал 2"]
```

**Решение:**
1. Станции → Проверьте `display_name` метрик
2. Должно совпадать ТОЧНО с `visibleMetrics`

---

### 🔴 Проблема 3: Неправильный unit или field

**Логи:**
```
  - Детали видимых метрик: [
    {id: 1, display_name: "Температура", field: "value", unit: "C"}
  ]
  🌡️ Температурных метрик: 0
```

**Решение:**
1. Станции → Измените `unit` с `"C"` на `"°C"`
2. Или измените `field` на что-то с "темп" или "temp"

---

### 🔴 Проблема 4: Backend не возвращает metric_X

**Логи:**
```
📊 Сырые данные (первые 3 записи):
  [0]: {timestamp: "2026-03-17T12:30:00Z"}  ← НЕТ metric_1, metric_2
```

**Решение:**
1. F12 → Network → `/api/history` → Response
2. Если там нет `metric_X` → проверьте backend
3. backend/src/routes/data.ts → getHistoryData
